# 🚀 AI-PR-Review-Assistant | AI 智能代码审计助手

> 基于 **DeepSeek-V3** 大模型驱动的生产级全栈自动化代码评审与安全审计平台。系统采用独创的**双阶段分析管线**，实现从远程 Git Diff 异步抓取、多维度质量加权评估，到行级漏洞深度诊断与代码闭环重写的全链路生态。

---

## 📐 系统设计分析

### 1. 分析准确性

- **双阶段强制管线**：Phase 1 语义分析 → Phase 2 代码审计，Phase 1 排在前面以确保模型在注意力最充足时完成业务理解
- **5 步推理链**：技术动作识别 → 业务动机推断（8 分类）→ 影响范围 → 风险评级 → 意图输出，每步有明确的判定标准
- **低温度确定性**：`temperature: 0.1` + `response_format: json_object`，确保同一 diff 多次审查结果一致

### 2. 上下文理解

- 以**完整 Unified Diff** 为核心输入，包含文件路径、hunk 头、行级变更，提供充分的结构化上下文
- 并行拉取 **GitHub PR 元数据**（标题、描述、作者、关联 Issue）作为业务上下文补充
- **22 项模块关键词映射表** 将文件路径自动翻译为业务模块名，增强上下文语义

### 3. 误报与漏报控制

- **"软约束 + 硬兜底"策略**：语义字段不加入 Schema `required`，避免字段缺失导致整体请求失败（防漏报）；通过自然语言强制声明 + 代码层自动接管弥补约束力
- **正则风险模式扫描**：10 项高危正则（SQL 注入、硬编码凭证、XSS 等）+ 7 项中危正则，即使模型完全不可用仍能捕获明显风险
- **六维质量评估标准**：安全/正确性/可维护性/架构/测试/影响范围，每维有具体可验证的判定标准（防误报）
- **三层容错**：Prompt 层 → 本地推理层 → Mock 降级层，三条路径全覆盖，杜绝漏报白屏

### 4. 响应速度

- PR 元数据与 Diff 获取**并行 fetch**，减少串行等待
- **CDN 优先**：Diff 拉取优先走 `patch-diff.githubusercontent.com`，回退才走 GitHub 主站
- 语义 + 审计**合并在一次 API 调用**完成（1RTT 而非 2RTT）
- Diff 超过 50000 字符**自动截断**，避免超长输入导致超时

### 5. 使用体验

- **双输入模式**：粘贴 PR 链接自动拉取 / 直接粘贴 Diff 文本
- **终端风暗色主题**：JetBrains Mono 等宽字体、行级绿/红高亮、红色连接线锚定评论到具体代码行
- **一键复制修复代码** + 已修复标记，打造"诊断→修复"闭环
- **优雅降级**：最差情况下返回高保真演示数据，前端无白屏无报错

### 6. 模型选择思路

选择 **DeepSeek-chat** 的核心理由：中文技术文档理解能力突出（审计结论需输出中文）、原生 JSON Mode 保证输出合法、OpenAI 兼容接口便于未来切换、成本显著低于 GPT-4/Claude。类型系统已预留 `AiModel: 'deepseek' | 'gemini'` 多模型架构。

### 7. 上下文获取方式

```
PR URL → parseGithubPr() 解析 → fetchWithTimeout() 拉取 .diff
       → cleanDiffContent() 八进制解码 → isValidUnifiedDiff() 校验
       → detectCodeBlocks() 结构识别 → 送入双阶段管线
```
同时通过 `undici ProxyAgent` 支持 `HTTPS_PROXY` 代理，确保受限网络环境中也能正常获取上下文。

### 8. 未来扩展方向

| 方向 | 内容 |
|------|------|
| **多模型协同** | 语义分析与审计分离到不同模型，或同 diff 多模型竞技，分歧项标"待人工确认" |
| **仓库级上下文** | 拉取变更文件的 main 分支完整内容，让模型看到修改前后全貌 |
| **规则+A.I.混合** | 团队自定义 YAML 规则引擎做确定性扫描（零误报），AI 补全模糊判断 |
| **CI/CD 集成** | GitHub App 自动评论到 PR 行、Webhook 触发、评分 < 阈值时阻断合并 |
| **增量审查** | 同 PR 多次 push 仅审查增量 diff，历史结果缓存复用 |
| **反馈闭环** | 追踪建议采纳率，反向优化 prompt 和规则 |

## ✨ 核心核心模块与特性

### 1. 🌐 异步网络层与远程 Git 深度集成
* **免授权抓取网关**：系统后端采用轻量级异步网络架构，支持一键粘贴 GitHub PR 链接，秒级拉取远程 Unified Diff 变更文本。
* **多文件标签化分类**：前端自动解析 Diff 树，将受影响的文件进行标签化隔离渲染，并高亮标记缺陷密度。

### 2. 🧠 独创的双阶段分析管线 (Phase 1 & Phase 2)
* **Phase 1：业务语义分析 (Business Intent)**
  AI 拒绝盲目扫描，而是 Step-by-Step 优先推断技术动作、划分系统模块，自动提炼出精准的**中文业务意图**，让审计具备全局视野。
* **Phase 2：强约束质量审计**
  通过前后端锁定的 Schema 结构化输出，聚焦**安全性、可读性、性能、容错性**四大维度进行加权评分，直观量化代码质量。

### 3. 🛠️ 行级智能诊断与代码重写闭环
* **微观架构诊断**：诊断精确到**具体文件和具体行号**，以交互式卡片形式深度剖析潜在隐患（如 V8 引擎去优化行为、未使用的泛型参数等）。
* **SUGGESTED REWRITE (修复方案)**：当场输出高可读性的修复补丁，支持一键复制代码（COPY CODE）与标记执行修复，打造“诊断-重写”的无缝闭环。

### 4. 🛡️ 工业级 Failover 容错降级架构
* **高拟真兜底保障**：考虑到网络抖动、海外大模型接口限流等极端不可抗力，后端路由设计了高拟真容错降级机制。一旦异常，在 **0.1 秒内自动捕获并无缝触发降级大屏**，确保系统在任何高并发、突发演示场景下的 100% 流畅可用。
